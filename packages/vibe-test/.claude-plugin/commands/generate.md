Use the **generate** skill to handle `/vibe-test:generate`.

Read `skills/generate/SKILL.md` and follow it end to end. The generate command:

1. Checks the blocking prereq — an audit-state must exist for the current scope. If missing, offer to run `/vibe-test:audit` first; never generate against an un-audited scope.
2. Inherits scope from the most recent audit. `--path <glob>` narrows (requires matching `audit-<hash>.json`). `--full` forces full-repo against `audit.json`.
3. Announces Pattern #13 complements (superpowers:test-driven-development → new-feature tests defer to TDD skill; playwright → E2E tests defer to Playwright's `--codegen typescript`; vibe-sec → elevate priority of flagged surfaces; vibe-doc → co-author TESTING.md sections).
4. For each audit gap in priority order: reads 2-3 similar existing tests (idiom sampling via `generator/idiom-matchers/`), composes a candidate test, runs env-var detection (`generator/env-var-scanner`), and assigns a confidence rating.
5. Routes per confidence: ≥0.90 auto-write to the detected test location with header comment `Generated by Vibe Test vX.Y.Z on <date>. Confidence: HIGH. Audit finding: #N.`; 0.70–0.89 stage in `.vibe-test/pending/` mirroring source tree with HEAD-hash recorded for branch-switch detection; <0.70 inline in chat with rationale for per-test accept/reject.
6. At accept time for staged tests: compares recorded HEAD to current HEAD. If different, surfaces the branch-switch warning — builder can `--force`, re-generate, or leave staged.
7. Skips tests already rejected with a reason in this project (`rejected.json`) or already covered (`accepted.json`) per G5.
8. Matches detected framework idioms (vitest / jest) + assertion style + fixture approach + test location per G6.
9. Honors scoped audit — generates only for that scope unless `--full` is passed.
10. Emits three output views in parallel: markdown (`docs/vibe-test/generate-<date>.md`), terminal banner, JSON sidecar (`.vibe-test/state/generate.json`).
11. Appends to `docs/test-plan.md` + updates `docs/TESTING.md` via handoff writers (H1, H2).
12. Offers CI stub opt-in once per project (H4). Declines recorded so future runs skip the prompt silently.
13. Writes session-log + friction-log (per-reject reasons) + project-state updates in the terminal phase.

**Not in this command (item #8 territory):** `--dry-run`, `--apply-last-dry-run`, rejection-pattern probe (3-consecutive-rejects pause + guidance prompt).

Every user-facing string respects `src/reporter/tier-adaptive-language.ts` verbosity knobs. JSON output is level-invariant.

Handoff line format is *"Run `/vibe-test:coverage` next. Run `/vibe-test:fix` if any tests fail."* (persona-adapted). Do **not** prescribe `/clear` between commands — Claude Code auto-compacts.
